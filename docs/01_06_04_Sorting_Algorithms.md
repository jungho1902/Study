# 정렬 알고리즘 (Sorting Algorithms)

**정렬(Sorting)**은 주어진 데이터 집합을 특정 기준(예: 오름차순, 내림차순)에 따라 순서대로 나열하는 과정입니다. 정렬은 데이터를 더 쉽게 검색하고 분석할 수 있게 만들어주므로, 컴퓨터 과학에서 가장 기본적이면서도 중요한 알고리즘 중 하나입니다.

다양한 정렬 알고리즘이 있으며, 각각의 장단점과 성능(시간 복잡도)이 다릅니다.

---

### 1. 버블 정렬 (Bubble Sort)

- **원리:** 서로 인접한 두 원소를 비교하여, 정해진 기준을 만족하지 않으면 자리를 교환하는 방식을 반복합니다. 한 번의 순회가 끝나면 가장 큰(또는 가장 작은) 원소가 맨 끝으로 이동하게 됩니다.
- **시간 복잡도:** `O(n²)`
- **특징:** 구현이 매우 간단하고 직관적이지만, 데이터가 많아질수록 성능이 급격히 저하되어 거의 사용되지 않습니다.

---

### 2. 선택 정렬 (Selection Sort)

- **원리:** 전체 데이터 중에서 가장 작은(또는 가장 큰) 원소를 찾아 맨 앞으로 보냅니다. 그 다음, 나머지 데이터 중에서 가장 작은 원소를 찾아 두 번째 위치로 보내는 과정을 반복합니다.
- **시간 복잡도:** `O(n²)`
- **특징:** 버블 정렬과 마찬가지로 구현은 간단하지만 비효율적입니다. 데이터 교환 횟수는 적다는 장점이 있습니다.

---

### 3. 삽입 정렬 (Insertion Sort)

- **원리:** 데이터를 하나씩 확인하면서, 각 데이터를 이미 정렬된 부분의 올바른 위치에 '삽입'하는 방식입니다. 손에 든 카드를 정렬하는 방식과 유사합니다.
- **시간 복잡도:** 평균 `O(n²)`, 최선 `O(n)`
- **특징:**
  - 데이터가 거의 정렬되어 있는 상태(nearly sorted)에서는 매우 효율적으로 동작합니다.
  - 데이터의 개수가 적을 때 다른 복잡한 정렬 알고리즘보다 빠를 수 있습니다.

---

### 4. 병합 정렬 (Merge Sort)

- **원리:** **분할 정복(Divide and Conquer)** 전략을 사용하는 대표적인 알고리즘입니다.
  1.  **분할(Divide):** 데이터 집합을 더 이상 나눌 수 없을 때까지(원소가 하나만 남을 때까지) 절반으로 계속 나눕니다.
  2.  **정복(Conquer) 및 결합(Combine):** 나누어진 부분 집합들을 다시 합치면서 정렬합니다. 두 개의 정렬된 부분 집합을 하나의 정렬된 집합으로 '병합(merge)'하는 과정이 핵심입니다.
- **시간 복잡도:** `O(n log n)` (최선, 평균, 최악 모두 동일)
- **특징:**
  - **안정 정렬(Stable Sort)**입니다. 즉, 값이 같은 원소들의 상대적인 순서가 정렬 후에도 유지됩니다.
  - 항상 `O(n log n)`의 시간 복잡도를 보장하여 안정적입니다.
  - 정렬을 위해 추가적인 메모리 공간(`O(n)`)이 필요하다는 단점이 있습니다.

---

### 5. 퀵 정렬 (Quick Sort)

- **원리:** 병합 정렬과 마찬가지로 **분할 정복** 전략을 사용합니다.
  1.  **분할(Divide):** 데이터 집합 내에서 하나의 기준점, 즉 **피벗(Pivot)**을 선택합니다.
  2.  피벗을 기준으로 피벗보다 작은 값들은 왼쪽, 큰 값들은 오른쪽으로 분할합니다 (파티셔닝).
  3.  **정복(Conquer):** 나누어진 왼쪽과 오른쪽 부분 집합에 대해 재귀적으로 이 과정을 반복합니다.
- **시간 복잡도:** 평균 `O(n log n)`, 최악 `O(n²)`
- **특징:**
  - 평균적으로 매우 빠른 성능을 보여 '퀵' 정렬이라는 이름이 붙었습니다.
  - 추가적인 메모리 공간을 거의 사용하지 않습니다 (`O(log n)`).
  - 피벗을 어떻게 선택하느냐에 따라 성능이 크게 달라집니다. 이미 정렬된 데이터를 피벗으로 선택하는 최악의 경우, 성능이 `O(n²)`로 저하될 수 있습니다.

파이썬의 내장 정렬 함수 `sort()`와 `sorted()`는 보통 **팀소트(Timsort)**라는 하이브리드 정렬 알고리즘을 사용합니다. 팀소트는 병합 정렬과 삽입 정렬의 장점을 결합하여 실제 데이터에서 매우 좋은 성능을 내도록 설계되었습니다.
